## 🛠️ Troubleshooting Common Challenges

### ❌ Issue: "Injection Failed" Error

**Symptoms**: Executor fails to inject despite successful target detection

**Solutions**:
1. Verify you're running as administrator (Windows) or with proper permissions
2. Check if antivirus is blocking the injection process
3. Ensure the target application is running before injection attempt
4. Update to the latest version of isaeva executor
5. Try the `--force-inject` command line flag

### ❌ Issue: Scripts Not Executing

**Symptoms**: Scripts load but don't execute or show errors

**Solutions**:
- Confirm script compatibility with current version
- Check console output for specific error messages
- Clear script cache: Settings → Advanced → Clear Cache
- Verify script syntax in the built-in editor
- Test with a known-working script from the hub

### ❌ Issue: High CPU/Memory Usage

**Symptoms**: System slowdown during operation

**Solutions**:
1. Lower the `cpu_limit` and `ram_limit` in configuration
2. Close unnecessary background applications
3. Enable "Low Resource Mode" in settings
4. Update graphics drivers
5. Reduce number of concurrent script executions

### ❌ Issue: Antivirus False Positives

**Symptoms**: Security software flags the executor as malicious

**Solutions**:
- Add installation folder to antivirus exclusions
- Whitelist the executable file specifically
- Use Windows Defender exclusively (disable third-party AV during use)
- Download only from official sources to ensure file integrity
- Verify file hash matches published checksums

### ❌ Issue: Mobile Installation Blocked

**Android Specific**:
- Enable "Install from Unknown Sources" in security settings
- Grant all requested permissions during setup
- Ensure Play Protect isn't blocking installation

**iOS Specific**:
- Trust the developer certificate after sideloading
- Re-sign the app if it expires (7-day cycle for free Apple ID)
- Check if AltStore/Sideloadly is properly configured
